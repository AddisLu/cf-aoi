行車紀錄檢視器 — log 一鍵跳到出錯的 code

載入 IP 產生的 _diag/<yyyyMMdd>.jsonl 或 incident_<ts>.json, 每筆 incident 會依其 src 欄位組成 VS Code 連結,點擊直接跳到出錯的源碼行。

純前端、完全離線;不上傳任何檔案。對應 IP flight_recorder 的 src 欄位(FR_RECORD_INCIDENT 巨集自動帶入)。

怎麼用

  1. 設定 repo 根目錄(你審查 code 那台機器上的 clone 路徑;預設 /Users/yourulyu/coding/cf-aoi)。
  2. 選檔載入 _diag/ 下的 .jsonl 或 incident_*.json(或直接貼上內容)。
  3. 每筆 incident 右側出現 ↗ 開啟源碼 → 點它,瀏覽器詢問後以 VS Code 開到該檔該行。
為何要設 repo 根目錄IP 跑在 Linux/Spark,但你常在 Mac 看 code,兩邊路徑不同。 所以 IP 只記機器無關的相對位置(如 ip/src/control_server.cpp:503);這裡用「你本機的根目錄」把它組成完整 vscode://file/… 連結。換機器只要改這個欄位。

運作原理

IP 出錯 → FR_RECORD_INCIDENT(kind, detail)
巨集自動帶入呼叫點 __FILE__:__LINE__(編譯期常數,零成本)
repo 相對化
incident JSON / jsonl 多一個 "src" 欄位
如 "src":"ip/src/control_server.cpp:503"(機器無關)
本檢視器 + repo 根目錄
組成 vscode://file/<root>/<src>
點擊 → VS Code 開到該檔該行

IP 端對應修改見 ip/src/diag/flight_recorder.{h,cpp}(src 參數 + FR_RECORD_INCIDENT 巨集 + repo_relative() 正規化)。 此功能屬「純觀測」,不碰計時/缺陷陣列 → 不破 bit-exact(不變式 16)。IP 改動目前 L1,待 Linux/Spark 重編 + 觸發 incident 驗證升 L3。

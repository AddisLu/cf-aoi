IP — GPU 缺陷檢測引擎

分散式架構的運算節點。接收影像,在 GPU 上執行 LCD 彩色濾光片缺陷檢測 (8 方向鄰域 DIV 比例式閾值),輸出結構化結果。

ip/ · C++ / CUDA · RTX 2080 Super(開發)→ DGX Spark GB10(生產) · 來源:docs/ip_程式完整說明.md

系統概述

核心策略:從 Reference/Demo 遷移 GPU 演算法,kernel 本體一字不改,只換 I/O 外殼。 IP 額外新增了 Reference 沒有的:CCL 收斂迴圈、canonical 排序、network-clean、多 ROI、行車紀錄、DIV-only 強制、pixel→μm。

為何「kernel 不改」是鐵律 11 個 AI/檢測 kernel 已逐行比對與 Demo byte-identical。改 kernel 本體會破壞已驗證的 bit-exact 與跨架構一致性; host wrapper(編排)可改。

GPU 檢測 pipeline — 12 階段

單張影像從進入到輸出,GpuPipeline::process_frame() 對每個 zone 跑完整流程。 紅框 = 決定性關鍵點。

1
影像進入 + 尺寸守衛 [1,16384] + FrameHeader magic/CRC 驗證 + FlightRecorder.set_scene
2
多 DetectRoi 裁切子影像(zone_rect);GPUMemoryManager 同尺寸重用 buffer
3
上傳:zero-copy mapped(GB10)/ pinned+async(x86);cudaEventRecord(start)
4
LSC 鏡頭暗角校正(可選,預設 off)
5
8 方向鄰域比較(DIV 比例式) — center/mean₈ > BTH(亮 255)/ < DTH(暗 128);Local Search ±1/±2 行消傾斜
6
CCL 連通分量標記 + 收斂迴圈 — atomicMin union,迴圈到 d_changed==0(typical 2-4 次)→ order-independent
7
Blob 分析 + 缺陷提取(sparse hash);過濾 size/aspect/density;上限 MAX_DEFECTS=10000
8
AI 分類過濾(gate: ai_on && ai_active,預設關)
9
cudaEventRecord(stop) + Synchronize → process_time_ms
10
主機端 canonical 排序 — std::sort 依 label→center_y→center_x→size → bit-exact
11
多 zone 聚合成 InspectionResult + FlightRecorder.record_frame(純觀測)
12
序列化輸出:ResultInfo.json/.xml + Defect_*.png + overlay;GlobalPos pixel→μm

可審查程式碼(真實源碼)

① DIV 比例式閾值 — 整數運算避開浮點

ip/src/gpu/cuda_kernels.cu:189–201審查:BTH/DTH 比例語意 + 整數縮放
// Integer threshold comparison (avoid floating point)
const int center_check = (center_val << 3) * 1000;   // center×8 對應 8 鄰域和
const int bth_scaled = __float2int_rn(BTH * 1000.0f);
const int dth_scaled = __float2int_rn(DTH * 1000.0f);

uint8_t result = 0;
if (center_check > neighbor_sum * bth_scaled) {
    result = 255;        // PointBright(亮缺陷)
} else if (center_check < neighbor_sum * dth_scaled) {
    result = 128;        // PointDark(暗缺陷)
}
d_binary[y * width + x] = result;   // 255/128/0 編碼供 CCL 依賴
DIV-only 等價等價於 center/mean₈ > BTH(亮)/ < DTH(暗), 與 legacy AlgorithmCompare="DIV" 嚴格相等。SUB(灰階差)無法不依賴背景灰階轉成比例 → IP 直接拒絕載入。

② CCL 收斂迴圈 — host wrapper(決定性關鍵)

ip/src/gpu/cuda_kernels.cu:1188–1199審查:單趟 race → 飄動;迴圈到不動點才 bit-exact
// host-side launch orchestration ONLY; __global__ kernel bodies unchanged.
// atomicMin's fixed point is unique & order-independent → loop until d_changed==0
{
    int h_changed = 1;
    const int MAX_CCL_ITERS = 1000;  // safety cap; typical 2-4
    for (int iter = 0; iter < MAX_CCL_ITERS && h_changed; ++iter) {
        cudaMemsetAsync(g_sparse.d_changed, 0, sizeof(int), stream);
        kernelFastCCLMerge<<<gridDim, blockDim, 0, stream>>>(
            d_binary, d_labels, width, height, g_sparse.d_changed);
        cudaMemcpyAsync(&h_changed, g_sparse.d_changed, sizeof(int),
                        cudaMemcpyDeviceToHost, stream);
        cudaStreamSynchronize(stream);
    }
}
遷移陷阱Reference 原版只跑單趟(lock-free union-find 未收斂)→ 缺陷數飄動(2761/2762/2763)。 從 Reference 重抄會覆蓋此迴圈,決定性就壞。

③ canonical 排序 — 把 race 順序變成全序

ip/src/gpu/gpu_pipeline.cpp:292–298審查:atomicAdd append 順序不定 → 排序後 bit-exact
std::sort(result.defects.begin(), result.defects.end(),
    [](const DefectInfo& a, const DefectInfo& b) {
        if (a.label    != b.label)    return a.label    < b.label;
        if (a.center_y != b.center_y) return a.center_y < b.center_y;
        if (a.center_x != b.center_x) return a.center_x < b.center_x;
        return a.size < b.size;
    });

關鍵不變式

Kernel 禁改cuda_kernels.cu / ai_kernels.cu 的 __global__ 本體 byte-exact 不改;host 編排可改。
同影像 bit-exact兩跑結果完全相同(CCL 收斂 + canonical 排序);--verify-deterministic 驗。
DIV-only只收 AlgorithmCompare="DIV";SUB 拒絕載入並報錯。
Pitch 爆量陷阱pitch 偏 1px → 30-50 缺陷;偏 4px → 561-1000;大偏差觸頂 10000。新面板先 FFT 估算。
AI 預設停用模型載入但 ai_active=false;缺陷標「待人工複核」進 DefectSort。--use-ai 才開。
block_dim 固定 16×16硬寫死,忽略 INI 的 [GPU] block_dim。實測 ~7.4ms/張,1 台 Spark 足夠。
行車紀錄純觀測計時區外、bench no-op、不破 bit-exact;incident:cuda_fatal / frame_validation / recipe_load / bad_json / uncaught_exception。

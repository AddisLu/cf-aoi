系統概述

從 Reference/cfaoi_phase1(測試套件:單 buffer、逐幀同步、無背壓)升級成生產等級: N-slot ring buffer + MrInfoEx 握手 + credit 背壓 + 狀態機 + TCP 控制面。

pylon / eBUS 嚴格分離cam_pylon.cpp 與(未來)cam_ebus.cpp 禁止互相 include — 不同廠商 SDK 可能全域狀態衝突。

③ MrInfoEx 握手結構(256B wire)

grab/src/rdma_common.h:45–54 ↗審查:IP 告知 ring 基底/rkey/n_slots/slot_size;static_assert 256B
struct MrInfoEx {
    uint64_t addr;       // N-slot ring 基底(cudaHostAlloc pinned)
    uint32_t rkey;       // 整塊 buffer RDMA 授權(一個 MR 涵蓋全部 slot)
    uint32_t len;        // = n_slots * slot_size
    uint32_t crc;        // 未使用(對齊舊 MrInfo)
    uint32_t n_slots;    // ring 深度(預設 4)
    uint32_t slot_size;  // = sizeof(FrameHeader) + max_payload
    uint8_t  pad[228];   // 對齊至 256 bytes
};
static_assert(sizeof(MrInfoEx) == 256, "MrInfoEx must be 256 bytes");
GB10 cudaHostAlloc(不變式)GB10(DGX Spark)RDMA 收圖不可用 nvidia_peermem(NVLink-C2C SoC GPU Bus ID 非標準 PCIe)。 IP 端 ring buffer 改用 cudaHostAlloc(Portable|Mapped),GPU 經 cudaHostGetDevicePointer 透過 NVLink-C2C(~900GB/s)讀寫。Grab 端 txbuf 為一般 host 記憶體,不受影響。

關鍵不變式(8 條)

1 · pylon/eBUS 分離禁止互相 include。
2 · FrameHeader 永不變256B,magic 0xA01CF00D,version 2,與 IP 同版。
3 · 無 UI全命令驅動(TCP 8100 或 CLI)。
4 · RoCE v2 必須RDMA NIC link_layer = Ethernet(非 IB)。
5 · MTU 區分相機 NIC 9000(GigE jumbo);RDMA NIC 不需 jumbo。
6 · GB10 禁 nvidia_peermem改 cudaHostAlloc(Portable|Mapped);IP 端改,Grab 無需改。
7 · N-slot + credit 背壓slot_id = seq % n_slots;poll_one() 背壓點。
8 · IP credit 順序不可換memcpy → push_blocking → post_recv 固定;先 post_recv 會掩蓋背壓。
